webpack的加载和打包指令

Posted on 2018-06-06

webpack

webpack是一款强大的模块加载器兼打包工具，它能把各种资源，例如JS(含JSX)、coffee、样式（含less/sass）、图片等都做为模块来使用和处理，优势如下：

  • Webpack是以commonJS的形式来书写，但对AMD/CMD的支持也很全面，方便旧项目进行代码迁移

安装webpack或者less最好不要安装全局，否则可能导致webpack的版本差异

在package.json中配置一个脚本，这个脚本用的命令是webpack，会去当前的node-modules下找对应的webpack名字，让其执行，执行的就是bin/webpack.js。web pack.js需要当前目录下有个叫webpack.config.js的文件。我们通过npm run build 执行的目录是当前文件夹的目录，所以会去当前目录下查找。

  • Babel 转义 es6 -> is5
  • Babel-preset-es2015

让bebel 具有解析es6语法的功能

  • 具有es7语法

Babel-preset-stage-0包含了-1 -2 -3 -4，-4是es6语法

  • 解析样式css

Css-loader 将css解析成模块，将解析的内容插入到style标签内（style-loader）

  • less, sass, stylus

css预处理语言，扩展了css语言

less-loader,less,css-loader,style-loader

sass-loader,stylus-loader

  • 图片的处理

File-loader url-loader 是依赖于file-loader的

  • 需要解析html插件

插件的作用是以我们自己的html为模版将打包后的结果自动引入到html中，产出到dist目录下。

  • Webpack-dev-server

实时显示效果，这里面内置了服务，可以帮我们启动一个端口号，当代码更新时，可以自动打包（在内存中打包），代码有变化就重新执行

  • 安装vue-loader vue-template-compiler(只有runtime.js情况下)

vue-loader：用来解析.vue文件

Vue-template-compiler：用来解析vue模版
